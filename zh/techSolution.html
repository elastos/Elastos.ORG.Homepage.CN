技术解决方案

  • 主链共识 ( PoW、DPoS、AuxPoW+DPoS )

    主链共识可以采用 Blockchain PoW、DPoS或混合 AuxPoW + DPoS 共识机制。AuxPoW 组件已于 2018 年在合作项目中成功完成，当时联合挖矿在 Bitmain 的矿池中投入使用。DPoS 在保护 Blockchain 方面起着同样重要的作用。混合共识中，PoW 矿工把区块哈希通过联合挖矿技术加入，而 DPoS 超级节点对其进行签名，从而创造了最终结果并防止分叉发生。 这种混合共识机制既利用 PoW 网络的巨大算力和功能，又利用 DPoS 来防止分叉，最终形成强大而安全的 Blockchain 。

  • DID 链

    DID 链采用 PoW 共识，可与 Blockchain 合并开采，以便利用 PoW 网络提供给 Blockchain 的相同安全性和散列能力。 DID（ 分布标识符 ）侧链向不受个人或第三方控制的用户、应用和设备发出完全分布的 ID ，可以被认为是一个自主和安全的识别系统。为每个用户提供数字身份所有权的独立证明，并使每个人能够以相同的无成本方式生成 DID 。 DID 可用于登录各种应用程序，也可用于存储个人数据和数字资产。从本质上讲， DID 链是运营机制的核心，为个人提供“拥有自己的数据”的权利和能力。

  • 数字资产 Token 链

    与其他公链（ 例如以太坊使用其主链向 DApps 发放自定义 ERC20 和 ERC721 Tokens ）不同，数字资产 Token 链将 Token 发布过程隔离在一个独特的侧链上。这种独特的 Token Sidechain 可以为在合作伙伴应用上构建的任何分布式应用程序发放可替代和不可替代的 Token 。该 Token 发布过程与用于可替代 Token 的相应 ERC20 标准和用于非可替代 Token 的 ERC721 标准并行地起作用。在该链的任何分布式应用程序都可以通过此链发出自己的自定义 Token 。由于其唯一的功能是向应用项目上发放 Token ，将有助于 Blockchain 链变得拥挤，采用 PoW 共识，并且还与 Blockchain 合并，后者又与比特币合并，因此获得比特币网络提供给主链的相同安全性和散列能力。请注意，此侧链仅有助于发布自己的稳定和安全的 Token 项目，并且没有自己的智能合约功能。

  • 智能合约兼容链

    自 2013 年成立以来，以太坊为可编程、分布式智能合约铺平了道路。此链能够移植现有的以太坊智能合约平台，专门用于处理用 Solidity 编写的智能合约的执行。开发人员无需学习新语言，如果精通 Solidity ，可以开始编写智能合约并将它们集成到部署在 Blockchain 上的 DApps 中。此外，如果现有智能合约 DApp 目前在以太坊上运行，它也可以轻松移植到该链上。智能合约兼容链将管理与以太坊兼容的智能合约的执行，并有助于发行 ERC20 和 ERC721 兼容 Token 。请注意，此链使用 DPoS 共识并访问相同的 DPoS 超级节点池以进行智能合约执行，与用于签署块的主链一样。

  • NEO 智能合约兼容链

    除了与以太坊智能合约兼容的侧链外，还可移植其他现有智能合约平台，以在其生态系统中运行。 NEO 智能合约兼容侧链允许开发人员编写和执行最初为 NEO 平台编写的智能合约。通过这个链，现有的 NEO 开发人员可以轻松迁移已经在 NEO 上编写的智能合约到这条侧链上，以及用他们熟悉的方式开发新的智能合约。请注意，最初，此侧链仍然采用 PoW 共识，并与主链联合挖矿，进而与比特币联合挖矿。如果需要将 PoW 的共识更改为 DPoS 以便更快地生成块，则可以轻松地执行转换。

  • 自定义侧链模板（ AuxPoW 或 DPoS ）

    除了使开发人员能够在其分布的应用程序基础架构上执行为以太坊和 NEO 区块链编写的智能合约之外，还提供自定义侧链模板，开发人员可以创建自己的自定义侧链。关于一致性算法，定制侧链可以通过主链的联合挖矿结构利用比特币网络的散列能力来利用工作量证明（ AuxPoW ）。或者，他们也可以选择实施委托证明（ DPoS ），即利用已经为 Blockchain 生态系统提供服务的 DPoS 节点。
    通过提供对预先建立的有组织节点集群的访问以保护侧链，自定义侧链模板将减少在其平台上构建的进入门槛。请注意，此模板仅指创建多个 NEO 智能合约兼容链或多个智能合约兼容链的功能。例如，当运行以太坊智能合约的 DApp 需要为自己创建额外的智能合约兼容链时，自定义侧链模板的用例就会出现。对于运行NEO 智能合约的 DApp ，自定义侧链模板提供相同的解决方案，侧链创建过程的具体细节尚未确定。这种可扩展性的解决方案，在理论和技术上可以创建尽可能多的智能合约兼容链、NEO 智能合约兼容链和数字资产 Token 链，以优化具体项目的整个生态系统的功能，从而提供高效的可扩展性解决方案。

  • Carrier

    Carrier 是一个分布式的点对点网络通信层，代表应用程序接管所有网络流量，以保护未来的数据和数字资产。这里重点跟踪亦来云的 Carrier，将支持 Windows 以补充其在 Linux 和 Mac 上的支持。

  • Hive 存储

    Hive 存储是一种基本的服务基础架构，可为具体 DApp 提供存储功能。 Hive利用标准的 IPFS / Cluster 开源项目进行一些必要的改进，同时保持与标准IPFS网络分离的完全独立的存储网络。典型的 IPFS 对等体是资源匮乏的程序，因为它在安装 IPFS 守护进程时消耗资源并减慢移动设备的速度。亦来云正在创建 Hive 项目，该项目使用 IPFS Cluster 作为亦来云 App 存储后端，可用于资源消耗低的场景。虽然该项目源于 IPFS 集群，但其运作与其前身相比有很大不同。Hive Cluster 维护一个大的 IPFS Pinset 用于共享，可以为众多虚拟 IPFS 对等服务，同时只需要一个运行真正的 IPFS 对等体。Hive 提供开源 IPFS 版本的所有服务，还具有与 Blockchain 生态系统中其他服务兼容的附加优势。这项服务有效地为开发人员提供了一个包罗万象的软件包。从本质上讲，Hive 允许开发人员在生态系统中构建他们的整个平台，从而提高生产力并增加网络的吸引力和广度。结合 DID 服务，Hive 使应用程序能够以分布式方式存储用户数据，以便数据仅供其所有者访问。

  • SPV 移动钱包

    SPV 移动钱包模块可快速帮助应用连接 Blockchain，并支持溶合上述不同的服务功能。
