技术解决方案

  • 亦来云主链共识 (AuxPoW + DPoS)

    亦来云 Blockchain 采用混合AuxPoW + DPoS 共识机制。 AuxPoW 组件于2018年8月成功完成，当时联合挖矿在 Bitmain 的矿池中投入使用。 亦来云共识机制的第二个组成部分 DPoS 在保护亦来云 Blockchain 方面起着同样重要的作用。 PoW 比特币矿工把 ELA 的区块哈希通过联合挖矿技术加入到比特币区块中参与挖矿，而 DPoS 超级节点对其进行签名，从而创造了最终结果并防止分叉发生。 这种混合共识机制既利用比特币网络的巨大功能，又利用 DPoS 来防止分叉，最终形成强大而安全的亦来云 Blockchain 。 一旦 DPoS 上线，它将为超出 Bitmain 范围的矿池和个人开放联合挖矿操作铺平道路，从而有助于增强强大且真正分布的网络。

  • 亦来云DID侧链

    DID 侧链于2018年7月正式上线，是第一款在亦来云上推出的产品。作为亦来云的首个侧链，它采用了 PoW 共识，并与亦来云合并开采，以便利用比特币网络提供给亦来云主链的相同安全性和散列能力。 DID（分布标识符）侧链向不受个人或第三方控制的用户、应用和设备发出完全分布的 ID ，可以被认为是一个自主和安全的识别系统。侧链为亦来云的每个用户提供数字身份所有权的独立证明，并使每个人能够以相同的无成本方式生成 DID 。 DID 可用于登录各种应用程序，也可用于存储个人数据和数字资产。从本质上讲， DID 侧链是运营机制的核心，为个人提供“拥有自己的数据”的权利和能力。

  • 亦来云Token侧链

    与其他公链（例如以太坊使用其主链向 DApps 发放自定义 ERC20 和 ERC721 Tokens）不同，亦来云将 Token 发布过程隔离在一个独特的侧链上。这种独特的 Token Sidechain 可以为在亦来云上构建的任何分布式应用程序发放可替代和不可替代的 Token 。该 Token 发布过程与用于可替代 Token 的相应 ERC20 标准和用于非可替代 Token 的 ERC721 标准并行地起作用。在亦来云上运行侧链的任何分布式应用程序都可以通过此 Token 侧链发出自己的自定义 Token 。由于其唯一的功能是向亦来云上的项目发放 Token ， Token 侧链将有助于防止亦来云主链变得拥挤，采用 PoW 共识，并且还与亦来云合并，后者又与比特币合并，因此获得比特币网络提供给亦来云主链的相同安全性和散列能力。请注意，此侧链仅有助于在亦来云上发布自己的稳定和安全的 Token 项目，并且没有自己的智能合约功能。

  • 以太坊（ETH）智能合约兼容侧链

    亦来云能够移植现有的 ETH 智能合约平台，以便在亦来云上执行。自2013年成立以来，以太坊为可编程、分布式智能合约铺平了道路。因此，为了轻松移植为以太坊编写的现有应用程序，亦来云正在开发一个侧链，专门用于处理用 Solidity 编写的智能合约的执行。利用这个侧链，开发人员无需学习新语言；相反，如果他们精通 Solidity ，他们可以开始编写智能合约并将它们集成到部署在亦来云上的 DApps 中。此外，如果现有智能合约 DApp 目前在以太坊上运行，它也可以轻松移植到亦来云上运行。以太坊智能合约兼容侧链将管理与以太坊兼容的智能合约的执行，并有助于发行 ERC20 和 ERC721 兼容 Token 。请注意，亦来云的以太坊侧链使用 DPoS 共识并访问相同的 DPoS 超级节点池以进行智能合约执行，与用于签署块的主链一样。

  • NEO 智能合约兼容侧链

    除了与以太坊智能合约兼容的侧链外，亦来云还将移植其他现有智能合约平台，以在其生态系统中运行。 NEO 智能合约兼容侧链允许开发人员编写和执行最初为 NEO 平台编写的智能合约。通过这个侧链，现有的 NEO 开发人员可以轻松迁移已经在 NEO 上编写的智能合约到这条侧链上，以及用他们熟悉的方式开发新的智能合约。请注意，最初，此侧链仍然采用 PoW 共识，并与亦来云主链联合挖矿，进而与比特币联合挖矿。如果需要将 PoW 的共识更改为 DPoS 以便更快地生成块，则可以轻松地执行转换。

  • 自定义侧链模板（AuxPoW 或 DPoS）

    除了使开发人员能够在其分布的应用程序基础架构上执行为以太坊和 NEO 区块链编写的智能合约之外，亦来云还提供自定义侧链模板，开发人员可以创建自己的自定义侧链。关于一致性算法，定制侧链可以通过亦来云的联合挖矿结构利用比特币网络的散列能力来利用工作量证明（ AuxPoW ）。或者，他们也可以选择实施委托证明（ DPoS ），即利用已经为亦来云生态系统提供服务的 DPoS 节点。
    通过提供对预先建立的有组织节点集群的访问以保护侧链，亦来云的自定义侧链模板将减少在其平台上构建的进入门槛。请注意，此模板仅指创建多个 NEO 侧链或多个以太坊侧链的功能。例如，当运行以太坊智能合约的 DApp 需要为自己创建额外的以太坊侧链时，自定义侧链模板的用例就会出现。对于运行 NEO 智能合约的 DApp ，自定义侧链模板提供相同的解决方案，侧链创建过程的具体细节尚未确定。在这个阶段，这个过程将反映起草提案给 CR ，因为新的侧链需要得到 CRC 理事会委成委员和社区的接受才能在亦来云上部署。通过这种安全、民主的过程，亦来云提供了一种水平可扩展性解决方案，在理论和技术上可以创建尽可能多的以太坊、NEO 和 Token 侧链，以优化整个生态系统的功能，从而提供高效的可扩展性解决方案。

  • 亦来云 Carrier

    亦来云 Carrier 是一个分布式的点对点网络通信层，代表应用程序接管所有网络流量，以保护未来的数据和数字资产。亦来云 Carrier 将推出一个新版本，主要关注两个主要领域：群发消息和文件传输。此版本将为亦来云 Carrier 平台带来许多功能和其他改进。此外，亦来云 Carrier 将支持 Windows 以补充其在 Linux 和 Mac 上的支持。

  • 亦来云Hive（亦来云存储）

    亦来云 Hive 是一种基本的服务基础架构，可为具体 DApp 提供存储功能。 Hive利用标准的 IPFS / Cluster 开源项目进行一些必要的改进，同时保持与标准IPFS网络分离的完全独立的存储网络。典型的 IPFS 对等体是资源匮乏的程序，因为它在安装 IPFS 守护进程时消耗资源并减慢移动设备的速度。亦来云正在创建 Hive 项目，该项目使用 IPFS Cluster 作为亦来云 App 存储后端，可用于资源消耗低的场景。虽然该项目源于 IPFS 集群，但其运作与其前身相比有很大不同。 亦来云 Hive Cluster 维护一个大的 IPFS Pinset 用于共享，可以为众多虚拟 IPFS 对等服务，同时只需要一个运行真正的 IPFS 对等体。 亦来云 Hive 提供开源 IPFS 版本的所有服务，还具有与亦来云生态系统中其他服务兼容的附加优势。这项服务当他们开始在亦来云上开发时有效地为开发人员提供了一个包罗万象的软件包。从本质上讲，亦来云 Hive 允许开发人员在亦来云生态系统中构建他们的整个平台，从而提高生产力并增加亦来云网络的吸引力和广度。结合亦来云的 DID 服务，亦来云 Hive 使应用程序能够以分布式方式存储用户数据，以便数据仅供其所有者访问。

  • 亦来云 手机移动钱包

    亦来云 手机移动钱包于12月初发布用于 Android 设备，并支持 ELA 以及 DID 侧链 Token 。未来，该钱包还将为 DPoS 超级节点选举提供投票机制支持。简而言之， SPV 移动钱包的功能将继续扩展，并将在 Android 和 iOS 平台上很好地发布。

  • 亦来云生态钱包( Elephant )

    亦来云生态钱包已于12月上架 App store 和 Google Play 。支持多种数字币种和亦来云生态伙伴币种。并且支持内置 DID 跨应用管理数字资产。其本地保管私钥，作为个人数字令牌，管理身份和资产。通过直接访问节点而非 SPV 与区块链交互，给用户带来更好的使用体验。
